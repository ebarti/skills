# Scoring Rubric Rules

How to score each dimension 0–10, combine into the weighted overall, and stay harsh. Default to skepticism: a typical untargeted resume lands 3–5, not 7.

## Be Harsh (read first)

- **Default skeptical.** Start each dimension assuming the resume is generic, then award points only for proof you can point to in the text. Absence of evidence = low score.
- **A typical untargeted resume is a 3–5 overall, not a 7.** "Looks fine" is not Strong. Strong (7–8) requires demonstrated JD coverage AND quantified results.
- **No benefit of the doubt.** If a must-have isn't stated, it's missing. If a result has no number and no qualified scope, it's a duty, not an achievement.
- **One fatal flaw caps the score.** Irrelevant to the JD, or a glaring red flag (typos, AI slop, missing all must-haves) → cap overall at 4 regardless of other dimensions.
- **Reserve 9–10.** Only for precise JD match + compelling quantified impact + zero red flags + ATS-clean. Most good resumes top out at 7–8.

## Scoring Each Dimension 0–10

Use this generic band, then the per-dimension anchors below.

| Score | Meaning |
|-------|---------|
| 0–2 | Absent / actively harmful |
| 3–4 | Weak, generic, mostly missing |
| 5–6 | Partial; basics only |
| 7–8 | Strong; minor gaps |
| 9–10 | Excellent; near-complete, no faults |

### 1. JD Match (30%)
- Count the JD's must-haves + key nice-to-haves. Score ≈ fraction covered **in the JD's own language, in the top third**, each backed by proof.
- 9–10: ~all must-haves + most nice-to-haves, mirrored up top. 7–8: most must-haves. 5–6: ~half, or present but buried/reworded. 3–4: a few keywords, generic. 0–2: wrong field / no overlap.

### 2. Achievement Strength (25%)
- Score by share of bullets that are CAR with a metric (%, $, time, volume, ranking) or qualified scope.
- 9–10: most bullets quantified, scale named. 7–8: several strong metrics. 5–6: a couple of numbers among duties. 3–4: mostly "responsible for"; 0–2: pure duty list.

### 3. Targeting & Focus (15%)
- 9–10: objective header mirrors the exact title; one theme top-to-bottom; off-target trimmed (5% rule applied). 7–8: clearly aimed, minor drift. 5–6: generic header, some off-target content. 3–4: "I'll do anything" feel. 0–2: aimed at a different role.

### 4. ATS Parseability (10%)
- 9–10: reverse-chronological, standard headings (Summary/Experience/Education/Skills), consistent dates, clean single-column, .docx/PDF. 5–6: mostly standard with risks (tables/columns/nonstandard headings). 0–2: functional format, graphics, unparsable file.

### 5. Red Flags (10%) — start at 10, deduct
- −3 typos/grammar errors; −2 unexplained gap or job-hopping not addressed; −2 clutter (Objective label, "References available", prep dates, phone, high school); −2 personal IDs / bias-inviting detail; −3 AI slop (generic, hallucinated, no specifics); −1 salary mentioned. Floor 0.

### 6. Language (10%)
- 9–10: tight bullets (<3 lines), power verbs + keyword nouns, active voice no "I", consistent tense/jargon, acronyms translated. 5–6: mixed (some filler/passive). 3–4: "responsible for", clichés, paragraphs. 0–2: dense, vague, inconsistent.

## Combining Into the Overall

**Formula:**
```
overall = JDmatch×0.30 + Achievement×0.25 + Targeting×0.15
        + ATS×0.10 + RedFlags×0.10 + Language×0.10
```
Result is already on the 0–10 scale. Round to one decimal, then map to the band. Apply the fatal-flaw cap (≤4) AFTER computing, if triggered.

### Worked Calculation
Scores: JD 4, Achievement 3, Targeting 5, ATS 7, Red Flags 8, Language 5.
```
4×0.30 = 1.20
3×0.25 = 0.75
5×0.15 = 0.75
7×0.10 = 0.70
8×0.10 = 0.80
5×0.10 = 0.50
-------------------
overall = 4.70 → Band 3–4? No: 4.7 rounds into 5–6 → "OK"
```
Note how weak JD Match + Achievement (the 55% that matters most) drag a resume with decent format down to barely "OK". That is the intended harshness.

## Checklist Failure → Deduction

Each failed book checklist item lowers its dimension's sub-score; a failure that touches multiple items compounds.

| Checklist failure | Hit dimension | Typical effect |
|-------------------|---------------|----------------|
| Must-haves absent / not in role's language | JD Match | −2 to −5 (dominant) |
| Duties instead of CAR; no numbers | Achievement | cap at 3–4 |
| Vague/me-centric objective; off-target content | Targeting | −2 to −4 |
| Non-RC format, nonstandard headings, bad file | ATS | −3 to −6 |
| Typos / unaddressed gaps / clutter / bias | Red Flags | per deduction table |
| Filler, passive voice, paragraphs, inconsistency | Language | −2 to −4 |

## Guidelines
- Always cite the missing must-have or absent metric in the critique — never just give a number.
- When two bands are plausible, choose the lower one.
- "Could be Strong with fixes" is still scored as what it is now, not its potential.

## Exceptions
- **CV roles (science/healthcare/academia):** length limits relax; do not penalize length under ATS/Targeting for genuine CVs.
- **True new grad / education-driven target:** Education-before-Experience is correct; don't flag under ATS or Red Flags.
- **No specific posting (database upload):** score JD Match against the common requirements of several current ads for the role.

## Quick Reference
| Rule | Summary |
|------|---------|
| Default skeptical | Untargeted resume = 3–5, not 7 |
| Fatal flaw cap | Irrelevant/major red flag → overall ≤4 |
| Reserve 9–10 | Precise match + quantified + clean + ATS only |
| Formula | Σ(score × weight); rounds to 0–10 band |
| Cite evidence | Name the missing must-have / absent metric |
