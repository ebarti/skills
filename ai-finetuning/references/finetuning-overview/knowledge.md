# Finetuning Overview Knowledge

Core concepts for understanding what finetuning is, its types, and how it compares to prompt-based methods (prompting and RAG).

## Overview

Finetuning adapts a pre-trained model to a specific task by adjusting some or all of its weights. Unlike prompt-based methods (which steer behavior via instructions, context, and tools), finetuning changes the model itself. It is most commonly used to improve instruction-following, output structure/format, and domain-specific quality.

## Key Concepts

### Finetuning

**Definition**: Further training a base model on task-specific data so it performs better on a target task.

**Key points**:
- Starts from a base model that already has *some* of the needed capabilities
- Goal: refine behavior, not teach the model from scratch
- Per the InstructGPT paper, finetuning often "unlocks" capabilities the base model already has

### Transfer Learning

**Definition**: Reusing knowledge gained from one task to accelerate learning on a related task. Finetuning is one way to do transfer learning.

**Key points**:
- Improves *sample efficiency* — fewer examples are needed than training from scratch
- Example: training legal QA from scratch may need millions of examples; finetuning a strong base may need only a few hundred
- Alternative: *feature-based transfer* (use a model to extract embeddings used by a downstream model)

### Self-Supervised Finetuning (Continued Pre-Training)

**Definition**: Finetuning on raw, unlabeled task-related text using the same self-supervised objective as pre-training.

**Use it when**: You have lots of cheap unlabeled domain text (e.g., raw legal documents, Vietnamese books) and want to cheaply prime the model before expensive supervised finetuning.

### Supervised Finetuning (SFT)

**Definition**: Training on (input, output) pairs — typically (instruction, response) — using high-quality annotated data.

**Key points**:
- Responses can be open-ended (summarization) or close-ended (classification)
- Also called *instruction finetuning*
- High-quality instruction data is expensive, especially when factual consistency or domain expertise is required

### Preference Finetuning (Alignment)

**Definition**: Training with reinforcement learning to maximize human preference, using comparative data.

**Data format**: `(instruction, winning_response, losing_response)`.

### Infilling Finetuning

**Definition**: Finetuning a model to fill in blanks (predict tokens given both left and right context), even if the base was pre-trained autoregressively.

**Useful for**: Text editing, code debugging.

### Long-Context Finetuning

**Definition**: Finetuning that extends a model's maximum context length (e.g., 4K → 16K tokens for Code Llama).

**Caveats**:
- Requires architectural changes (e.g., adjusting positional embeddings)
- Harder than other finetuning types
- The result may degrade on shorter sequences

### Distillation

**Definition**: Finetuning a smaller model to imitate a larger model, using data generated by the large model.

**Why**: Small finetuned models are cheaper, faster, and can outperform much larger out-of-the-box models on a specific task. (Grammarly's Flan-T5 beat a GPT-3 variant 60x its size on text editing with 82K examples.)

## Terminology

| Term | Definition |
|------|------------|
| Base model | Pre-trained (and often post-trained) model used as the finetuning starting point |
| Pre-training | Initial self-supervised training on large unlabeled corpora |
| Post-training | Any training the model developer does after pre-training (often SFT + alignment) |
| SFT | Supervised finetuning — training on labeled (input, output) pairs |
| PEFT | Parameter-efficient finetuning — finetune a small fraction of weights |
| Sample efficiency | How well a model learns from a small number of examples |
| Continued pre-training | Self-supervised finetuning on task-related raw text |
| Distillation | Training a small model to imitate a large one |

## How It Relates To

- **Prompt engineering**: Steers behavior without weight changes; cheaper, faster, the first thing to try.
- **RAG**: Provides external knowledge at inference time without weight changes; addresses *information* gaps.
- **Finetuning**: Changes weights; addresses *behavior* gaps (form, style, structure, instruction following).

## Finetuning vs Prompting vs RAG

| Aspect | Prompting | RAG | Finetuning |
|--------|-----------|-----|------------|
| Changes weights? | No | No | Yes |
| Adds external knowledge? | No | Yes | No (internalizes from data) |
| Up-front cost | Low | Medium | High |
| Maintenance burden | Low | Medium | High |
| Best for | Quick iteration | Information gaps | Behavior/form gaps |
| Inference complexity | Unchanged | Increased (retrieval pipeline) | Unchanged |
| Development complexity | Low | Medium | High |

## Finetuning vs RAG: The Core Heuristic

> **Finetuning is for form. RAG is for facts.**

- **Information-based failures** (factually wrong, outdated, missing private knowledge) → RAG
- **Behavior-based failures** (correct but irrelevant, wrong format, wrong style, broken syntax) → Finetuning
- Both at once → Start with RAG (easier, larger boost), then add finetuning if needed

Per Ovadia et al. (2024): RAG with the *base* model often outperforms RAG with a finetuned model — finetuning can degrade orthogonal capabilities.

## Common Misconceptions

- **Myth**: "General-purpose models are always weak at domain-specific tasks, so I must finetune."
  **Reality**: Strong general-purpose models often beat specialized ones. GPT-4-0314 (zero-shot) beat BloombergGPT on financial benchmarks (87.15 vs 75.07 on FiQA F1).

- **Myth**: "Finetuning always reduces hallucinations."
  **Reality**: Finetuning can *worsen* hallucinations if data quality is low.

- **Myth**: "Finetuning lets me skip prompting work."
  **Reality**: You still need an evaluation pipeline, annotation guidelines, and prompt-quality discipline. Prompt experiments are stepping stones for finetuning.

- **Myth**: "Finetuning replaces RAG."
  **Reality**: They are complementary. The two combined improved MMLU scores 43% of the time over RAG alone.
