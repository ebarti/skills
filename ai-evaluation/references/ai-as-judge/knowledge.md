# AI as a Judge Knowledge

Core concepts for using AI models to evaluate other AI models.

## Overview

AI as a judge (also called LLM as a judge) uses an AI model to evaluate other AI outputs. This is one of the most common evaluation methods in production today, with 58% of evaluations on LangChain's platform done by AI judges in 2023. An AI judge is not just a model — it is a system combining a model, a prompt, and sampling parameters.

## Key Concepts

### AI Judge

**Definition**: An AI model used to evaluate the outputs of other AI models.

A judge takes a generated response (and optionally a question, reference, or second response) and produces a score, classification, or preference label. Often it can also explain its decision, which helps with auditing.

**Key points**:
- Works without reference data (usable in production)
- Can score on any criteria you define in the prompt
- Strong models correlate well with humans (GPT-4 reached 85% agreement with humans on MT-Bench, higher than human-human agreement of 81%)

### Three Evaluation Modes

**Definition**: The three common ways to apply an AI judge.

1. **Pointwise (quality of one response)** — Given (question, answer), score the answer.
2. **Reference-based** — Given (question, reference answer, generated answer), judge equivalence or similarity.
3. **Pairwise comparison** — Given (question, answer A, answer B), pick the better one. Used for preference data, test-time compute, and ranking.

### Scoring Systems

**Definition**: How the judge expresses its verdict.

- **Classification** — good/bad, relevant/irrelevant/neutral. LLMs do best with this.
- **Discrete numerical** — typically 1 to 5. Wider ranges perform worse.
- **Continuous numerical** — e.g., 0.0 to 1.0 for similarity degree. Hardest for LLMs.

LLMs are generally better with text than numbers; classification beats numerical scoring.

### Specialized Judges

**Definition**: Smaller models trained for specific judgment tasks.

- **Reward model** — takes (prompt, response), outputs a quality score. Used in RLHF. Example: Cappy (360M params).
- **Reference-based judge** — compares generated response to one or more references. Example: BLEURT (similarity), Prometheus (1–5 score given a rubric).
- **Preference model** — takes (prompt, response 1, response 2), outputs which is preferred. Examples: PandaLM, JudgeLM.

### Self-Evaluation (Self-Critique)

**Definition**: A model judging its own output.

Subject to self-bias, but useful as a sanity check and for nudging the model to revise. If a model rates its own response as bad, that is a strong signal.

## Terminology

| Term | Definition |
|------|------------|
| AI judge | Model + prompt + sampling params used to evaluate outputs |
| Self-bias | Judge favors responses generated by itself |
| Position bias / first-position bias | Judge favors the first option in a pairwise comparison |
| Verbosity bias | Judge favors longer responses regardless of quality |
| Recency bias | Human counterpart — favoring the answer seen last |
| Spot-checking | Evaluating only a subset of responses to control cost |
| Reward model | Specialized judge: (prompt, response) → score |
| Preference model | Specialized judge: (prompt, A, B) → which is better |

## How It Relates To

- **Evaluation criteria**: AI judges score against criteria like faithfulness, relevance, coherence, toxicity — but criteria definitions vary across tools.
- **Comparative evaluation**: Pairwise judges produce preference data used to rank models.
- **Post-training alignment**: Preference judgments feed RLHF and DPO pipelines.
- **Production guardrails**: Many teams gate user-visible responses on a judge's verdict.

## Common Misconceptions

- **Myth**: An AI judge is just a model.
  **Reality**: It is a system — model + prompt + sampling parameters. Change any of them and you get a different judge.

- **Myth**: AI judges are objective and reproducible.
  **Reality**: They are probabilistic. The same judge on the same input can return different scores across runs.

- **Myth**: Faithfulness scores from MLflow, Ragas, and LlamaIndex are interchangeable.
  **Reality**: Each uses different prompts and scoring scales (1–5, 0/1, YES/NO). Scores are not comparable.

- **Myth**: The judge must be stronger than the model being judged.
  **Reality**: Judging is often easier than generating. Smaller specialized judges can outperform large general-purpose ones for specific criteria.

- **Myth**: A consistent judge is an accurate judge.
  **Reality**: A judge can be consistently wrong. Consistency is necessary but not sufficient.
